# Novel IDE 主题变量规范 (Theme Tokens)

本文档规定了 `theme-tokens.ts` 中定义的 CSS 变量的作用及使用场景。整个应用的所有组件必须**只**消费这些变量来定义颜色，以保证在切换 `sepia` / `light` / `dark` 主题时能获得完美一致的体验。

## 基础背景 (Backgrounds)

控制应用的主要结构背景层级。

- `--bg-main`: **最底层的背景色**。用于整个页面的 body 背景、或者最外层的大容器背景。它通常也是最暗/最灰的颜色，以便衬托上面的卡片。
- `--bg-panel`: **卡片或面板的背景色**。用于处于 `--bg-main` 之上的内容区域（例如居中显示的工作区面板、Scene 卡片、设置弹窗主体等）。在浅色/护眼模式下通常是纯白或最亮的颜色。
- `--bg-sidebar`: **侧边栏背景色**。用于应用侧边栏、目录树结构，它通常比 `--bg-main` 略亮，或者处于两者之间。

## 交互背景 (Interactive Backgrounds)

控制交互元素的底色，不涉及主题品牌色。

- `--bg-hover`: **通用悬停背景色**。用于鼠标悬停在列表项、透明按钮、或者是普通卡片上时的底色。
- `--bg-active`: **通用激活背景色**。用于列表中当前被选中的项、或者处于 pressed/active 状态的元素。（*新增，解决原本悬停和选中色彩拉不开层次的问题*）
- `--bg-input`: **输入组件/次级模块背景**。用于 `input`、`textarea` 内部底色，或卡片内部的“次级容器”（例如 Scene 卡片内的“场景摘要框”的底色）。通常带有一定的灰度/遮罩感。

## 边框 (Borders)

- `--border-color`: **标准边框色**。用于划分区域（分隔线、卡片边框、输入框边框等）。应当相对柔和。
- `--border-color-hover`: **悬停加强边框色**。用于按钮或卡片被悬停时，提供比 `--border-color` 具有更高对比度的边框反馈。（*新增，解决了过去未定义该变量导致的“黑边框”问题*）

## 文本与排版 (Typography)

- `--text-main`: **主文本色**。用于正文内容、标题、重要的状态标签等高对比度文本。
- `--text-secondary`: **次级文本色**。用于摘要内容、辅助说明文本、副标题等，对比度低于主文本。
- `--text-muted`: **弱化文本色**。用于占位符 (placeholder)、图标默认颜色、非常次要的序号或提示信息。
- `--text-inverse`: **反色文本**。用于需要深色底（例如高对比度的主题色按钮）上的文字。通常是纯白（浅色模式下）或纯黑（深色模式下）。

## 品牌与重点强调 (Accent & Brand)

用于关键操作（如“创建”、“保存”）及高亮状态。

- `--accent-main`: **主题强调色**。应用的核心品牌色（例如护眼模式下的橙褐色，明亮模式下的蓝色）。用于 Primary 按钮的背景、选中文本的高亮、或者激活状态的 Tab 下划线。
- `--accent-bg`: **强调色透明底色**。通常是 `--accent-main` 的 10%~15% 透明度版本。常用于高亮某一个块状区域而不显得突兀。
- `--accent-text`: **强调文字色**。用于链接、重点强调的数值。为了保证可读性，它通常比 `--accent-main` 更深（Light模式下）或更亮（Dark模式下）。

## 特殊业务模块 (Specialized)

这部分属于高定组件专用变量，仅在特定场景使用：

- **Toolbar (`--toolbar-bg`)**: 工具栏底色，通常带有一定的透明度以配合毛玻璃 (`backdrop-blur`) 效果。
- **Prompt (`--prompt-bg`, `--prompt-border`)**: 大语言模型对话气泡或输入框使用的底色。
- **Agent (`--agent-bg`)**: 智能体相关区域专有色。
- **Editor 系列**:
  - `--editor-canvas-bg`, `--editor-preview-bg`: 编辑器正文背景。
  - `--editor-shell-bg`, `--editor-head-bg`: 编辑器外壳和头部（支持半透明过渡）。
  - `--editor-gutter-bg`: 编辑器行号槽背景。
- **Source 系列**: 源码/代码块区域背景色 (`--source-bg`) 与文本色。

## 最佳实践与避坑 (Pitfalls)

1. **不要硬编码颜色**：避免在 `class` 中使用类似于 `bg-gray-100` 或 `text-[#333]` 这种绝对颜色。必须全部映射为 `bg-[var(--bg-panel)]`。
2. **谨慎使用 Opacity 叠加**：如果你觉得 `--bg-panel` 太亮，**不要**叠加 `bg-black/5` 这样的工具类，这会在 Dark 模式下产生“脏色”。应当考虑使用 `--bg-input` 或要求增加新的语义变量。
3. **边框反馈的正确做法**：实现 Hover 边框时，应始终写为 `border-[var(--border-color)] hover:border-[var(--border-color-hover)]`，而不要依赖浏览器的隐式回退机制。
